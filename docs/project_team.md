# Project Portfolio Assistant - Team Composition

## Project Overview
**Project Name:** Portfolio Assistant in Python  
**Project Leader:** Alessio (Admin user)  
**Purpose:**  
Development of an internal portfolio management tool for tracking and analyzing personal investment data, integrating with Telegram for natural language interaction, and supporting reporting on portfolio allocation and performance.

---

## Team Composition

### 1. Alessio – Project Leader
- Defines strategic objectives, oversees progress, and approves final deliverables.
- Maintains ownership of the GitHub repository and cloud deployment.
- Primary system administrator (super user).

### 2. Luigi – Senior DevOps Expert
- Specializes in setting up the CI/CD pipelines, versioning, and deployment automation.
- Ensures the environment structure is compatible with GitHub Actions and cloud deployment.
- Defines branching strategy, testing automation, and environment setup (dev, test, prod).

### 3. Laura – Senior Data Model Analyst
- Designs the data model and entity-relationship (E-R) diagrams.
- Defines data integrity rules, constraints, and optimization for performance and maintainability.
- Works with the product owner and architect to ensure scalability and consistency of data persistence.

### 4. Mike – Senior Solution Architect
- Expert in both cloud and on-premises deployment.
- Identifies and selects the best-fit architecture and services to minimize TCO (Total Cost of Ownership).
- Defines the infrastructure for database, backend, and integrations (Telegram, GenAI).

### 5. Claudio – Senior Product Owner
- Responsible for defining product features and translating business requirements into user stories.
- Ensures clarity of acceptance criteria for development and testing.
- Maintains functional backlog and prioritization for the development phases.

---

## Collaboration Principles
- **Code repository:** GitHub (main branch protection enabled)
- **CI/CD:** Automated testing and deployment pipeline (Luigi)
- **Documentation:** Stored in Markdown under `/docs`
- **Communication channel:** Telegram bot integration for internal notifications and testing
- **Release Management:** Versioned with semantic versioning (vX.Y.Z)
